# 微信小程序发布指南

## 📱 版本说明

微信小程序有三个版本：
- **开发版**：开发者工具中预览，仅开发者可见
- **体验版**：测试人员可体验，通过二维码访问
- **正式版**：审核通过后发布，所有人可访问

---

## 🚀 发布流程

### 1. 准备工作

#### 1.1 检查 AppID
确保已在 `project.config.json` 中配置正确的 AppID：
```json
{
  "appid": "你的微信小程序AppID"
}
```

#### 1.2 编译生产版本
```bash
# 编译小程序生产版本
pnpm build:weapp

# 编译后的文件在 dist/ 目录
```

#### 1.3 检查包体积
```bash
# 查看主包大小
du -sh dist/

# 主包限制：2MB
# 整个小程序所有分包大小不超过 20MB
```

## 🎯 发布体验版

### Step 1: 打开微信开发者工具
```bash
# 启动微信开发者工具
# 打开项目目录：/workspace/projects
```

### Step 2: 上传代码
1. 点击右上角「上传」按钮
2. 填写版本号（如：1.0.0）
3. 填写项目备注（如：修复登录问题，优化排行榜）
4. 点击「上传」

### Step 3: 设置体验版
1. 登录[微信公众平台](https://mp.weixin.qq.com)
2. 进入「开发」→「开发版本」
3. 找到刚上传的版本
4. 点击「选为体验版本」

### Step 4: 添加体验者
1. 进入「管理」→「成员管理」→「项目成员/体验成员」
2. 搜索微信号
3. 添加为「体验成员」

### Step 5: 体验者访问
1. 微信扫描体验版二维码
2. 打开后即可体验

**注意**：体验版最多 100 个体验成员

---

## 🏆 发布正式版（上线）

### Step 1: 上传代码
同体验版 Step 2

### Step 2: 提交审核
1. 登录[微信公众平台](https://mp.weixin.qq.com)
2. 进入「管理」→「版本管理」→「开发版本」
3. 点击「提交审核」
4. 填写审核信息：
   - 功能页面描述
   - 用户登录页面截图
   - 游戏玩法页面截图
   - 选择服务类目：游戏 > 棋牌/益智

### Step 3: 等待审核
- 审核时间：1-7 个工作日
- 审核结果会通过微信通知

### Step 4: 发布上线
1. 审核通过后，进入「审核版本」
2. 点击「发布」
3. 填写发布说明
4. 点击「确认发布」

**注意**：
- 每天只能提交 1 次审核
- 审核通过后可随时发布
- 发布后所有人可访问

---

## 📋 审核注意事项

### 1. 内容规范
- ❌ 禁止含有色情、暴力、赌博等内容
- ❌ 禁止诱导分享、诱导关注
- ❌ 禁止虚假宣传
- ✅ 必须有明确的隐私政策
- ✅ 必须有用户协议

### 2. 功能要求
- ✅ 游戏必须有退出功能
- ✅ 必须有用户协议和隐私政策
- ✅ 游戏内支付必须使用微信支付
- ✅ 必须有防沉迷系统（如有未成年人）

### 3. 技术要求
- ✅ 必须使用 HTTPS
- ✅ 必须适配 iPhone X 及以上机型
- ✅ 必须适配暗黑模式（iOS）
- ✅ 必须适配各种屏幕尺寸

### 4. 隐私合规
- ✅ 必须收集用户隐私授权
- ✅ 必须提供注销账号功能
- ✅ 必须明示数据收集用途

## 🔧 常见问题

### Q1: 审核被拒怎么办？
**常见原因**：
- 隐私政策不完善
- 用户协议缺失
- 游戏内容违规
- 功能描述不符

**解决方法**：
1. 根据审核意见修改
2. 重新提交审核
3. 修改后第二天再次提交

### Q2: 体验版无法打开？
**检查项**：
- AppID 是否正确
- 是否添加为体验成员
- 体验版是否已选为体验版本
- 小程序是否已发布过（首次发布需要审核）

### Q3: 正式版发布后如何更新？
**更新流程**：
1. 修改代码
2. 上传新版本（版本号 +1）
3. 提交审核
4. 审核通过后发布
5. 用户会自动更新到最新版本

### Q4: 如何回滚版本？
**回滚流程**：
1. 进入「版本管理」→「线上版本」
2. 点击「回退」
3. 选择要回退的版本
4. 确认回退

**注意**：只能回退到最近 5 个版本

### Q5: 包体积超过 2MB 怎么办？
**解决方法**：
1. 使用分包加载
2. 压缩图片
3. 移除未使用的代码
4. 使用 CDN 加载大文件

## 📝 版本号规范

### 语义化版本号
格式：`主版本号.次版本号.修订号`

示例：
- `1.0.0` - 正式版发布
- `1.0.1` - Bug 修复
- `1.1.0` - 新增功能
- `2.0.0` - 重大更新

### 发布建议
- **正式版发布**：主版本号不变，次版本号 +1
- **紧急修复**：修订号 +1
- **重大更新**：主版本号 +1
