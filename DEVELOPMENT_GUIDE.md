## 🎯 正确的开发方式

### 重要提示：不要直接预览根目录！

这个项目是 **Taro 项目**，不是原生小程序项目，所以：

### ❌ 错误的做法
- 直接在微信开发者工具中打开项目根目录
- 直接点击"预览"或"上传"
- 期望能直接运行

### ✅ 正确的做法

#### 第一步：启动开发服务器

在项目根目录下运行：

```bash
pnpm dev
```

这会启动 Taro 开发服务器，自动编译和热更新。

#### 第二步：在微信开发者工具中打开

1. 打开微信开发者工具
2. 选择 **"导入项目"**
3. 项目目录选择：**`/workspace/projects/dist`**（注意是 dist 目录，不是根目录！）
4. AppID 选择你的测试号或小程序 AppID
5. 点击 **"导入"**

#### 第三步：开始开发

- 修改 `src/` 目录下的代码
- Taro 会自动重新编译
- 微信开发者工具会自动刷新

## ☁️ 云函数部署

### 方法一：微信开发者工具（推荐）

1. 确保 `pnpm dev` 正在运行
2. 在微信开发者工具中（打开的是 dist 目录）
3. 点击顶部 **"云开发"** 按钮
4. 开通云开发环境
5. 右键 `cloudfunctions/` 下的文件夹
6. 选择 **"上传并部署：云端安装依赖"**

### 方法二：云开发控制台（最可靠）

1. 访问：https://console.cloud.tencent.com/tcb
2. 按 `QUICK_DEPLOY.md` 中的步骤操作
3. 手动复制云函数代码

---

## ⚠️ 注意事项

1. **始终在 `src/` 目录下写代码**，不要直接修改 `dist/` 目录
2. **运行 `pnpm dev` 后再打开微信开发者工具**
3. **微信开发者工具打开的是 `dist/` 目录**，不是项目根目录
4. **云函数代码在 `cloudfunctions/` 目录**，需要单独部署

## 🆘 常见问题

**Q: 微信开发者工具报错 "app.json not found"？**
A: 确保你打开的是 `dist/` 目录，不是项目根目录，并且已经运行了 `pnpm dev`

**Q: 修改代码后不生效？**
A: 确保 `pnpm dev` 正在运行，Taro 会自动重新编译

**Q: 云函数上传失败？**
A: 参考 `QUICK_DEPLOY.md`，使用云开发控制台手动部署
