# 真机预览与部署后测试指南

## 问题背景

真机预览（微信开发者工具扫码）和部署后开发版小程序可能出现差异，导致测试结果不一致。

## 差异原因分析

### 1. 代码版本差异
- **真机预览**：使用本地最新代码，实时更新
- **部署后**：使用构建后的代码，版本固定

### 2. 环境变量差异
- **真机预览**：使用本地开发环境变量
- **部署后**：使用服务器环境变量

### 3. 缓存问题
- **真机预览**：缓存较少，经常刷新
- **部署后**：缓存可能保留旧版本数据

### 4. 网络环境差异
- **真机预览**：本地网络环境
- **部署后**：生产网络环境

### 5. 资源加载时机差异
- **真机预览**：冷启动，首次加载
- **部署后**：可能有热缓存

## 解决方案

### 1. 使用开发工具页面

我已经为你创建了一个**开发工具页面**，可以在真机上查看和管理环境信息。

#### 访问方式
在首页**长按标题** "海海拼图大作战" 即可进入开发工具页面（仅开发环境可见）。

#### 功能说明
- 📦 **环境信息**：查看当前环境类型、版本号、构建时间、API 地址
- 💾 **缓存管理**：清除图片缓存、应用缓存、用户数据、所有缓存
- 🔍 **调试工具**：刷新信息、查看日志（需要在开发者工具查看）

### 2. 测试流程

#### 真机预览测试

1. **启动本地开发环境**
   ```bash
   coze dev
   ```

2. **微信开发者工具扫码预览**

3. **进入开发工具页面**
   - 长按标题 "海海拼图大作战"
   - 查看环境信息，确认环境类型为 `DEVELOPMENT`

4. **清除缓存**
   - 点击 "清除所有缓存"
   - 确保使用最新代码

5. **测试功能**
   - 测试所有功能，确保正常工作
   - 记录环境信息（版本号、构建时间）

6. **保存测试结果**
   - 截图保存环境信息
   - 记录测试用例和结果

#### 部署后测试

1. **部署代码到服务器**
   ```bash
   pnpm build
   # 上传代码到微信小程序后台
   ```

2. **扫码进入开发版**

3. **进入开发工具页面**
   - 长按标题 "海海拼图大作战"
   - 查看环境信息，确认环境类型为 `PREVIEW` 或 `PRODUCTION`

4. **清除缓存**
   - 点击 "清除所有缓存"
   - 确保使用最新代码

5. **对比环境信息**
   - 对比版本号是否一致
   - 对比构建时间是否一致
   - 对比 API 地址是否正确

6. **测试功能**
   - 按照相同的测试用例测试
   - 对比测试结果是否一致

### 3. 确保一致性的最佳实践

#### ✅ 正确做法

1. **统一测试流程**
   - 真机预览和部署后使用相同的测试用例
   - 每次测试前先清除缓存
   - 记录详细的环境信息和测试结果

2. **版本号管理**
   - 每次部署前更新版本号
   - 在开发工具页面确认版本号
   - 确保真机预览和部署后版本号一致

3. **环境变量检查**
   - 在开发工具页面查看 API 地址
   - 确保使用正确的环境（开发/测试/生产）
   - 不要在开发环境使用生产 API

4. **缓存管理**
   - 每次部署后清除所有缓存
   - 遇到问题时先尝试清除缓存
   - 使用缓存版本控制（已实现）

5. **日志收集**
   - 关键操作添加详细日志
   - 在开发者工具查看日志
   - 记录错误信息和堆栈

#### ❌ 错误做法

1. **不清除缓存直接测试**
   - 可能使用旧版本代码
   - 导致测试结果不准确

2. **不检查环境信息**
   - 无法确认当前环境
   - 可能测试了错误的环境

3. **不统一测试流程**
   - 真机预览和部署后测试用例不同
   - 无法对比结果

4. **依赖本地缓存**
   - 本地缓存可能在其他环境不存在
   - 导致测试结果不一致

### 4. 常见问题排查

#### 问题 1：音效延迟

**症状**：部署后音效播放有延迟，真机预览正常

**排查步骤**：
1. 进入开发工具页面，确认音效预加载是否执行
2. 查看控制台日志，确认 `preloadSounds()` 是否在 LoadingPage 调用
3. 清除缓存后重新测试

**解决方案**：
- 已在 LoadingPage 提前初始化音效
- 确保代码版本一致

#### 问题 2：图片加载失败

**症状**：进入关卡后短暂显示加载失败，然后图片才出现

**排查步骤**：
1. 进入开发工具页面，查看图片缓存大小
2. 查看控制台日志，确认图片加载状态
3. 清除图片缓存后重新加载

**解决方案**：
- 已增加图片加载重试机制
- 已增加详细的错误日志
- 已实现离线模式支持

#### 问题 3：关卡进度丢失

**症状**：退出后再进入从第1关开始，进度丢失

**排查步骤**：
1. 进入开发工具页面，查看用户数据缓存
2. 查看控制台日志，确认 `updateHighestLevel` 调用
3. 检查云端同步状态

**解决方案**：
- 已增加详细的日志输出
- 已确保本地存储优先
- 已添加同步失败提示

### 6. 工具和脚本

#### 查看环境信息
```typescript
import { getEnvInfo, logEnvInfo } from '@/config/env'

// 获取环境信息
const env = getEnvInfo()
console.log('环境类型:', env.type)
console.log('应用版本:', env.version)

// 打印详细环境信息
logEnvInfo()
```

#### 管理缓存
```typescript
import {
  clearAllCache,
  clearAppCache,
  logCacheInfo,
  formatCacheSize,
  getCacheSize
} from '@/utils/cache'

// 查看缓存信息
logCacheInfo()

// 清除所有缓存
clearAllCache()

// 清除应用缓存（保留用户数据）
clearAppCache(true)

// 获取缓存大小
const size = getCacheSize()
console.log('缓存大小:', formatCacheSize(size))
```
