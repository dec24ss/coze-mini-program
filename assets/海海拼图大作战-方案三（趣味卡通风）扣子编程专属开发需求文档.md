# 海海拼图大作战-方案三（趣味卡通风）扣子编程专属开发需求文档

**文档用途**：100%适配扣子低代码编程平台，全流程可视化落地微信小游戏，优先通过平台内置能力实现设计效果，仅核心逻辑补充极简兼容代码，无需专业开发能力即可完美还原，适配微信小程序生态规则。

**适配场景**：扣子低代码平台开发微信小游戏，750rpx基准尺寸（扣子/微信小程序原生适配单位，100%兼容不同屏幕）

**核心风格**：高饱和卡通、海洋趣味元素、活泼动态交互、强视觉记忆点

---

## 前置：扣子编程适配核心原则

本方案完全贴合扣子低代码平台的开发特性，所有设计均遵循以下规则，确保零障碍落地：

1. **组件优先**：所有元素优先用扣子内置组件实现，减少自定义代码，最大化利用可视化配置能力

2. **样式标准化**：所有样式采用扣子支持的CSS属性，单位统一用rpx（微信小程序/扣子原生自适应单位），避免不兼容属性

3. **动效轻量化**：动效优先用扣子内置动画组件实现，复杂动效拆解为平台支持的序列帧/基础动画，规避微信小程序性能限制

4. **数据极简管理**：所有状态（关卡、排名、设置）用扣子内置「全局变量/页面变量+本地存储」实现，无需复杂后端开发

5. **微信生态适配**：完全贴合扣子对微信小程序的原生适配规则，规避包体积、权限、接口调用的限制

## 一、基础设计规范（扣子编程适配版）

### 1.1 色彩系统（扣子直接填色，无修改）

|色彩类型|色值|应用场景|
|---|---|---|
|主色调1（活力橙）|#FF9500|开始游戏按钮、选中态、数字高亮、徽章|
|主色调2（海洋蓝）|#007AFF|关卡/设置按钮、图标、拼图边框、高亮背景|
|主色调3（草绿）|#34C759|排行榜按钮、成功提示、绿植装饰|
|辅助色1（沙滩黄）|#F5DEB3|背景纹理、设置按钮、底部装饰|
|辅助色2（贝壳粉）|#FFB6C1|装饰元素、图标点缀、气泡|
|中性色1（深灰）|#333333|正文文字、标题阴影、图标描边|
|中性色2（浅灰）|#E5E5EA|未解锁态、分隔线、背景浅层次|
|中性色3（透明白）|rgba(255,255,255,0.8)|弹窗背景、卡片遮罩|
扣子适配提示：颜色直接在扣子样式面板的「颜色选择器」中输入十六进制色值即可，无需额外调整。

### 1.2 字体规范（扣子兼容优化）

|文字类型|字体方案（扣子优先）|字号（rpx）|字重|颜色|应用场景|
|---|---|---|---|---|---|
|主标题|优先用**PNG图片替代**（卡通字体做成透明底图片），备选扣子内置「思源黑体加粗」|72rpx|加粗|海洋蓝/活力橙|游戏名称、页面大标题|
|副标题|同主标题，备选思源黑体|48-56rpx|常规|深灰|页面小标题、关卡名称|
|按钮文字|扣子内置「思源黑体圆角版」|40-44rpx|半粗|白色/深灰|所有按钮文案|
|正文/提示|扣子内置「思源黑体」|32-36rpx|常规|深灰/浅灰|说明文字、加载提示、排名信息|
|数字/关卡号|优先PNG图片替代，备选思源黑体加粗|56-64rpx|加粗|主色调轮换|关卡号、计时器、排名数字|
扣子适配提示：扣子不支持外部网络字体加载，特殊卡通字体必须用PNG透明底图片实现，避免真机不显示、版权问题。

### 1.3 图标规范（扣子素材库适配）

1. 造型：全量图标采用「圆润描边+填充」风格，边角无尖锐，线条粗细统一4rpx

2. 尺寸：按钮内图标96×96rpx，列表/装饰图标64×64rpx，迷你图标48×48rpx

3. 格式：优先SVG格式（缩放不失真），备选@2x PNG格式，所有素材背景透明

4. 核心图标库（必须按主题设计）：

    - 功能类：拼图块、奖杯、灯泡（提示）、眼睛（原图）、雪花（冻结）、齿轮（设置）、喇叭（音效）、铃铛（震动）、锁（未解锁）

    - 装饰类：小鱼、海浪、贝壳、螃蟹、气泡、闪光、浪花

5. 命名规范：`页面_元素_状态_尺寸`，例如`btn_灯泡图标_96rpx.png`，方便扣子素材库分类管理

### 1.4 全局变量配置（扣子直接新建）

提前在扣子「全局变量」面板新建以下变量，全页面通用，无需重复配置：

|变量名|变量类型|默认值|核心用途|
|---|---|---|---|
|currentLevel|数字|1|当前选中/游玩的关卡号|
|maxLevel|数字|1|用户已解锁的最高关卡号|
|userMaxLevel|数字|0|用户已通关的最高关卡|
|loadingProgress|数字|0|加载页进度百分比|
|loadedCount|数字|0|加载的图片资源计数|
|soundOn|布尔|true|音效开关状态|
|vibrateOn|布尔|true|震动开关状态|
|gameTime|数字|0|游戏计时器秒数|
|hintCount|数字|0|提示道具剩余次数|
|originalCount|数字|0|原图道具剩余次数|
|freezeCount|数字|0|冻结道具剩余次数|
扣子适配提示：变量变更时同步绑定「本地存储」动作，重启小程序数据不丢失。

## 二、各页面精细化设计需求（扣子编程分步落地版）

### 2.1 加载页（Loading Page）

【扣子页面创建】新建页面命名`loading`，设置为小程序启动页，页面背景透明

【核心容器】添加「全屏flex容器」，布局：垂直居中、水平居中，内边距80rpx，宽高100%

【分步搭建&扣子配置】

1. 背景层（分层嵌套容器，扣子支持层级管理）

|    层级|    组件选型|    样式配置|    动效配置（扣子内置动画）|
|---|---|---|---|
|    1（底层）|    全屏容器|    宽高100%，线性渐变背景#E6F4FF→#007AFF|    无|
|    2（中层）|    3个图片组件|    上传分层海浪PNG，宽100%，高度分别为600rpx/400rpx/200rpx，定位页面底部|    循环位移动画：X轴从-750rpx到0，时长10s，缓动linear，循环播放，实现海浪流动|
|    3（顶层）|    图片组件|    上传沙滩纹理PNG，宽100%，高度800rpx，定位页面底部|    无|
|    4（装饰）|    图片组件|    螃蟹、贝壳PNG，定位沙滩右下角，尺寸120×120rpx|    轻微缩放循环动画，时长2s|
2. 标题区

    - 组件：图片组件，上传「海海拼图大作战」卡通标题PNG（透明底）

    - 样式：宽600rpx，高度240rpx，margin-bottom 320rpx，层级5

    - 动效：页面加载完成后，添加缩放动画（0.8倍→1倍，时长300ms，缓动ease-out）

3. 进度条区

    - 容器：flex水平容器，宽600rpx，高度120rpx，margin-bottom 80rpx，层级5

    - 进度条容器：圆角矩形容器，宽600rpx，高度60rpx，背景#E6F4FF，圆角30rpx，相对定位

    - 进度条填充：圆角矩形容器，**宽度绑定全局变量{{loadingProgress}}%**，高度60rpx，背景#007AFF，圆角30rpx，绝对定位left 0，背景叠加波浪PNG遮罩

    - 进度数字：文本组件，绝对定位right 0，字号48rpx，加粗，颜色#FF9500，**内容绑定{{loadingProgress}}%**

    - 小鱼动画：图片组件，上传卡通小鱼PNG，宽120rpx，高度120rpx，**left绑定{{loadingProgress * 6}}rpx**（容器宽600rpx，1%=6rpx），添加±5°循环旋转动画，实现尾部摆动

4. 提示文字区

    - 容器：flex垂直容器，水平居中，层级5

    - 主提示：文本组件，字号36rpx，颜色#333333，**内容绑定「正在加载图片… {{loadedCount}}/10」**

    - 副提示：文本组件，字号32rpx，颜色#E5E5EA，内容「首次加载可能需要几秒钟」，margin-top 40rpx，左侧添加问号气泡图片组件

【事件&逻辑（扣子可视化配置）】

1. 页面加载时：触发「设置全局变量」，loadingProgress=0、loadedCount=0

2. 添加定时器：每500ms执行1次，loadedCount+1、loadingProgress=loadedCount*10，直到loadedCount=10

3. 条件判断：当loadingProgress=100时，触发「页面跳转」，跳转到首页`home`

【扣子避坑提示】

- 全屏容器必须设置宽高100%，避免不同屏幕出现白边

- 分层素材不超过5层，避免扣子渲染卡顿

- 定时器总时长不超过10s，规避微信小程序内存限制

### 2.2 首页（Home Page）

【扣子页面创建】新建页面命名`home`，页面背景透明

【核心容器】全屏flex容器，垂直布局，水平居中，上下内边距300rpx，宽高100%

【分步搭建&扣子配置】

1. 背景层

    - 主背景：全屏容器，线性渐变#E6F4FF→#FFFFFF，层级1

    - 海浪动画层：图片组件，上传半透明海浪PNG，宽100%，定位页面底部，添加循环位移动画（同加载页），层级2

    - 漂浮装饰：8-10个图片组件（拼图块、小鱼、贝壳），随机分布页面边缘，尺寸80×80rpx，添加上下浮动循环动画（幅度20rpx，时长2s），层级3

2. 标题区

    - 主标题：图片组件，上传「海海拼图大作战」卡通标题PNG，宽600rpx，高度240rpx，margin-bottom 40rpx，层级4

    - 副标题：文本组件，内容「拖拽碎片，完成拼图」，字号36rpx，颜色#E5E5EA，margin-bottom 200rpx，层级4

3. 按钮组（垂直排列，间距40rpx，层级4）

|    按钮名称|    组件选型|    样式配置|    图标+文字|    点击事件|
|---|---|---|---|---|
|    开始游戏|    按钮组件|    宽600rpx，高160rpx，圆角48rpx，线性渐变#FF9500→#FFB700，白色4rpx描边|    左侧96rpx拼图图标（白色）+ 右侧44rpx白色文字|    页面跳转→游戏页`game`，同时设置全局变量currentLevel=1|
|    关卡选择|    按钮组件|    宽600rpx，高160rpx，圆角48rpx，线性渐变#007AFF→#409EFF，白色4rpx描边|    左侧96rpx贝壳图标（白色）+ 右侧44rpx白色文字|    页面跳转→关卡选择页`level`|
|    排行榜|    按钮组件|    宽600rpx，高160rpx，圆角48rpx，线性渐变#34C759→#6FE788，白色4rpx描边|    左侧96rpx奖杯图标（白色）+ 右侧44rpx白色文字|    页面跳转→排行榜页`rank`|
|    设置|    按钮组件|    宽600rpx，高160rpx，圆角48rpx，线性渐变#F5DEB3→#FFFFE0，深灰4rpx描边|    左侧96rpx齿轮图标（深灰）+ 右侧44rpx深灰文字|    弹出设置弹窗（同页面弹窗容器，无需新建页面）|
【动效配置】

- 所有按钮：触摸时添加下沉动画（translateY 4rpx，阴影模糊从8rpx→4rpx，时长150ms），松开回弹

- 页面入场：所有按钮从下往上滑动入场（偏移100rpx→0，时长200ms，错开100ms触发）

【扣子避坑提示】

- 按钮触控区域默认全屏宽，避免误触

- 弹窗用同页面容器实现，不用新建页面，减少扣子页面数量限制

### 2.3 关卡选择页（Level Select Page）

【扣子页面创建】新建页面命名`level`，页面背景同首页

【核心容器】全屏flex垂直容器，顶部导航+信息条+滚动容器（关卡网格），宽高100%

【分步搭建&扣子配置】

1. 顶部导航（通栏750rpx，高度120rpx，margin-top 40rpx）

    - 左侧：图片组件（返回箭头，海洋蓝，96×96rpx），点击事件→页面跳转回首页

    - 中间：文本组件（选择关卡，56rpx站酷快乐体图片/思源黑体加粗，深灰）

    - 右侧：图片组件（三点菜单，浅灰，96×96rpx，可选功能）

2. 信息条（宽700rpx，高度100rpx，margin-top 40rpx）

    - 容器：圆角矩形容器，背景#E5E5EA，圆角32rpx，轻微阴影

    - 文本：内容绑定「最高关卡：第{{userMaxLevel}}关 | 已解锁：第{{maxLevel}}关」，字号40rpx，深灰，水平居中

3. 关卡网格（滚动容器，3列网格布局，列间距32rpx，行间距40rpx，margin-top 80rpx）

    - 关卡卡片尺寸：220rpx×220rpx，拼图块异形造型，圆角24rpx

    - 已解锁关卡：

        - 样式：背景色海洋蓝/活力橙/草绿轮换，白色4rpx描边

        - 内容：中间56rpx白色卡通数字（关卡号），下方32rpx橙色文字「可挑战」

        - 事件：点击→设置全局变量currentLevel=对应关卡号，跳转游戏页`game`

        - 动效：触摸时上浮16rpx+轻微旋转1°，时长200ms

    - 未解锁关卡：

        - 样式：背景#E5E5EA，无描边

        - 内容：中间96rpx浅灰锁图标，无文字

        - 事件：点击无动作，或弹出「通关前序关卡解锁」提示

    - 选中态：选中的关卡添加8rpx活力橙描边+闪光动画

4. 底部装饰：同首页卡通海浪层，定位页面底部

【逻辑配置】

- 页面加载时：从本地存储读取maxLevel、userMaxLevel，更新全局变量

- 关卡卡片显示：通过「条件渲染」，当关卡号≤maxLevel时，显示已解锁样式，否则显示未解锁

【扣子避坑提示】

- 关卡网格必须放在「滚动容器」内，适配关卡数量增加后的上下滑动

- 条件渲染用扣子内置「显示/隐藏」动作，不用自定义代码

### 2.4 排行榜页（Ranking Page）

2. 个人信息卡（宽700rpx，高度200rpx，margin-top 40rpx）

    - 容器：圆角矩形容器，线性渐变#007AFF→#409EFF，白色4rpx描边，圆角40rpx，轻微外发光

    - 内容：左侧96rpx铜色奖杯图标，中间文本「我的排名：第{{userRank}}名」（48rpx白色），下方小字「最高关卡：第{{userMaxLevel}}关」（36rpx透明白）

3. 排行榜列表（宽700rpx，可滚动容器，margin-top 40rpx）

    - 表头：宽700rpx，高度100rpx，背景#007AFF，圆角16rpx，3列flex布局，文本「排名/用户/关卡」（40rpx白色，平均分布）

    - 列表项（扣子内置列表组件，循环渲染）：

        - 自己的行：条件渲染，背景#007AFF，所有文字白色，头像替换为主角卡通头像

    - 交互：触摸行时，背景变为#F5DEB3，上浮4rpx

4. 底部按钮（宽600rpx，高度120rpx，margin-top 80rpx）

    - 样式：圆角矩形，线性渐变#FF9500→#FFB700，白色4rpx描边，圆角48rpx

    - 内容：44rpx白色文字「返回首页」

    - 事件：点击跳转回首页

【扣子避坑提示】

- 排行榜数据用扣子内置「云数据库」存储，无需自建后端，直接调用微信小程序开放能力获取用户昵称头像

- 列表用扣子循环组件实现，不用手动添加每一行

### 2.5 设置页（弹窗式，同首页实现）

【实现方式】首页内嵌套弹窗容器，默认隐藏，点击设置按钮时显示，无需新建页面

【核心容器】

- 遮罩层：全屏容器，背景rgba(0,0,0,0.3)，点击时隐藏弹窗，层级10

- 弹窗主体：圆角矩形容器，宽600rpx，高度600rpx，背景rgba(255,255,255,0.9)，白色4rpx描边，圆角40rpx，垂直居中，层级11

【分步搭建&扣子配置】

1. 弹窗标题区

    - 左上角：文本组件「设置」，56rpx卡通字体/思源黑体加粗，深灰

    - 右上角：图片组件（关闭X图标，海洋蓝，96×96rpx），点击时隐藏弹窗

2. 设置项（垂直布局，间距80rpx，弹窗垂直居中）

|    设置项|    左侧元素|    右侧组件（扣子内置开关组件）|    事件配置|
|---|---|---|---|
|    音效|    96rpx喇叭图标（海洋蓝）+ 44rpx深灰文字「音效」|    宽160rpx，高80rpx，圆角40rpx；开启态背景#007AFF，关闭态#E5E5EA，滑块白色圆形|    开关切换时，设置全局变量soundOn=开关状态，同步保存到本地存储|

```JavaScript
// 扣子编程拼图位置判断代码块，入参：拖拽组件、目标格子、正确格子ID
Component({
  methods: {
    checkPuzzlePosition: function(dragComp, targetGrid, correctGridId) {
      // 获取组件坐标
      const dragRect = dragComp.getBoundingClientRect();
      const gridRect = targetGrid.getBoundingClientRect();
      const allowOffset = 40; // 允许吸附偏差
      // 判断位置&正确性
      const isInGrid = dragRect.left >= gridRect.left - allowOffset && dragRect.right <= gridRect.right + allowOffset && dragRect.top >= gridRect.top - allowOffset && dragRect.bottom <= gridRect.bottom + allowOffset;
      const isCorrect = targetGrid.id === correctGridId;

      if (isInGrid && isCorrect) {
        // 正确：吸附锁定
        dragComp.setStyle({ left: gridRect.left + 'px', top: gridRect.top + 'px', position: 'absolute' });
        dragComp.setDisabled(true);
        this.triggerEvent('puzzleCorrect', { gridId: correctGridId });
      } else {
        // 错误：回弹
        dragComp.setStyle({ left: dragComp.dataset.initialLeft + 'px', top: dragComp.dataset.initialTop + 'px' });
        this.triggerEvent('puzzleError');
      }
    }
  }
})
```

## 六、扣子编程常见问题&解决方案

|常见问题|解决方案|
|---|---|
|拖拽组件真机不灵敏|扩大拖拽组件触控区域，最小48rpx×48rpx，关闭容器滚动穿透|
|动画真机卡顿|单页面同时播放的动画≤5个，关闭非必要循环动画，用序列帧替代复杂CSS动画|
|字体真机不显示|所有特殊卡通字体用PNG图片替代，不使用网络字体|
|重启后数据丢失|变量变更时，同步触发扣子「本地存储」动作，保存到wx.setStorageSync，页面加载时读取|
|小程序包体积超限|大素材放到扣子云存储，用网络链接加载，本地仅保留核心小素材|
|拼图位置判断不准|调整代码块中的allowOffset偏差值，最大不超过40rpx，避免误判|
